## 1. ⚠️ READ THIS FIRST — three live hazards

### 1a. The squash-merge orphan trap has bitten TWICE. It is armed again right now.

**Rule: after a PR merges, NEVER push to that branch again. Cut a fresh branch
from `origin/main`.**

- Round 1: PR #43 merged, work continued on `rc/RC-3d-realism-land` → recovered as #44.
- Round 2: PR #44 merged, work continued on the same branch again → recovered as #47.
  That stale branch predated PR #45 and merging it would have **silently reverted**
  the /acoperisuri SEO titles in both locales. Caught by diffing CONTENT, not
  commit counts — a squash merge makes every branch commit look "missing" even
  when its content is already on main.

**Always** verify with `git diff origin/main <branch> --stat` before opening a PR.

### 1b. There is uncommitted work sitting on a branch with an OPEN PR

## 5. The 3D house — the whole story

This is where the last week went, and where the project's frustration lives.

### 5.1 Why there is a 3D model at all
The homepage hero is a house that **builds itself** — pieces fly in and assemble
over 2.2 seconds, mapped to the company's real service phases (foundation →
walls → roof → openings → details). It is the one thing a photograph can never
do, and it is the entire justification for the 3D. **Protect the build
animation above all else.**

### 5.2 How the 3D work is organised (converged over ~10 sessions — keep it)
1. **A separate Claude Code terminal** runs the Blender agent (model: Fable 5),
   so the main session's credits go to site wiring. It reads `BLENDER-AGENT.md`
   at startup. Reconnect procedure: open Blender → N-panel → BlenderMCP →
   **Start Server** (port 9876) → `cd ~/Projects/rapidconstruct-web && claude`.
   Its memory is **files, not chat** — `claude --resume` is only for the literal
   old conversation.
2. **Exactly ONE task file in the repo root**, named `BLENDER-TASK-*.md`.
   Completed ones move to `docs/blender-tasks-done/` (23 archived). Two files in
   the root once made the agent re-run a stale task.
3. **Options-as-renders before building** any aesthetic change. The owner picks
   from pictures, then we build. This has saved the project twice (4 roof
   options; 3 de-barn looks). Guessing and shipping cost a full revert cycle.
4. **The main session verifies every export independently.** The agent's report
   is a claim, not proof. Phase-bucket against the component's real regexes,
   check every MUST-NOT-REGRESS item, audit AO/texCoord, then dedup + webp, then
   a browser render **including a dragged angle**.

### 5.3 The laws (all bought with pain — do not re-derive)
- **Verify in the REAL renderer.** Blender's viewport lies. Run the site and
  screenshot it headless.
- **Five-angle + close-in + mobile-drag + ray-scan** before every export. The
  owner finds defects by dragging on a phone; hero-distance shots miss them.
  (Born from: a see-through roof hole; scenery lying flat.)
- **Verification law:** if your check constructs the input it tests for, it
  proves nothing. (Born from a fake-UA probe that "proved" a Lighthouse fix that
  had never once fired.)
- **Exported-JSON law:** occlusion/texCoord wiring is invisible in Blender's
  viewport. Always dump the per-material texCoord table from the exported glb.
  This bug shipped three times and was hand-patched twice.
- **Never instance structural pieces** — an instanced batch animates as ONE
  piece, destroying the one-by-one build.
- **Draw calls (~330 node ceiling), not bytes, are the perf limit.** Profiled:
  ~480 draws/frame; the hero is draw-call-bound at only ~24k triangles.
- Imported assets: re-orient to Z-up, bake scale, verify world bbox z-span.
- `img.pack()` after any pixel edit; UVs + AO vertex-colour on any textured mesh.
- `| tail` eats exit codes — use `cmd > /tmp/x 2>&1; echo $?`.
- **Save the .blend before and after every experiment.** One sprint ended with
  hours of work living only in Blender's RAM.

### 5.4 The skill
`~/.claude/skills/blender-buildings/SKILL.md` — 719 lines, auto-loads in the
Blender session. Holds proven recipes: dark metal-tile / standing-seam roofs,
cheap edge realism (bevel + weighted normals), trim sheets, phase-naming
conventions for the build animation, the glTF slimming pipeline, and now the
ranked fake-tells list. **Anything proven and reusable gets promoted into it via
`skill-creator`** so it is never re-researched.

### 5.5 Where the model stands
`public/models/house.glb` — **1.04 MB, 307 nodes, `extensionsRequired: none`,
2 instanced batches.** Ship pipeline: `gltf-transform dedup` → `webp` (~25%
saving), both proven safe on three 0.185's stock GLTFLoader. Keep a raw copy
before compressing.

What is in it today: jerkinhead (clipped-gable) roof, no truss, dark vertical
timber cladding on gables + entry, warm greige walls, warm near-black roof,
modular pantile rhythm, real eaves (soffit + fascia + baked rafter tails),
rebuilt porch canopy, profiled windows with recessed reveals, hollow shell walls
in 155 course segments, full-house baked AO atlas, masonry+metal fence
(instanced, 7 modules), modeled tree, instanced snow guards. Build animation:
2.2 s, ~16 pieces airborne, blueprint-blue until 55% of each piece's flight.

Site-side rendering lives in `src/components/HouseBuildScene.tsx` (582 lines,
heavily commented with the reasoning behind every number): ACES tone mapping at
exposure 1.18, warm key sun with 512² shadows, cool fill, back rim, self-hosted
HDRI at `environmentIntensity` 0.55, ContactShadows, and a `rested` flag that
stops shadow updates once the house settles (that was the drag-lag fix).

### 5.7 What three research sprints established (so nobody repeats them)

Full logs in `BLENDER-NOTES.md` and `docs/blender-tasks-pending/round3-log.md`.

**Killed hypotheses** — do not revisit:
- "The materials are flat colours with no textures." **False.** The shipped glb
  has 24 materials, 13 images, 7 baseColor-textured materials, roof normal +
  roughness maps. This came from a stale 2026-07-15 scene dump that is now marked
  `⛔ STALE — DO NOT USE` at the top of `BLENDER-NOTES.md`.
- **Anisotropy 8** — measured indistinguishable. Refuted.
- **KTX2 / Basis** — the transcoder payload is 570 KB, larger than the 109 KB it
  saves at our scale. **Parked** until textures exceed ~2 MB.
- **AgX tone mapping** — measured 24% *less* saturated. Backwards.

**Established facts:**
- The "realistic vs 1.5 MB" trade is **fake**. `gltf-transform meshopt` + `webp`
  gets stage D to ~861 KB and projects to ≈1.2–1.35 MB at 1024² textures —
  under the gate. The unlock is meshopt, not KTX2.
  ⚠️ Tooling gotcha: gltf-transform *decompresses* gltfpack meshopt — keep the
  pipeline gltf-transform-only, or run gltfpack last.
- **Teardown of sites that look great**: lusion.co ships 6.6 MB of textures;
  bruno-simon.com ships ~1 MB of draco+basis glbs with **zero realtime lights**
  (all baked). Nobody realtime-lights 300 nodes at 1 MB. **We were the outlier.**
- **The ranked lever list**, by realism gained per hour:
  1. Golden-hour lighting rig + warm window glow — **0 bytes, 0 draws**
  2. Camera: level verticals, fov ≈30, polar clamp ≥55° — **0 bytes, 0 draws**
  3. Baked GI lightmap, crossfaded on `rested` — +53 KB, **−165 draws at rest**
  4. Real ground/context — the single worst region by measurement
  5. meshopt+webp ship pipeline — −455 KB
  6–9. Roof tile profile, hedge/tree, plaster scan, entourage
- **`scripts/realism-score.mjs`** compares a render against the owner's photos
  per region and emits a score. Baselines: current 50, A 50, B 50, C 50, D 52
  (roof 73 / wall 39 / ground 37→45). **Three stages of pure material work moved
  the score by zero.** That is the empirical case for why the pivot is right and
  why texture work is deferred.

**Everything in levers 3–9 is deferred**, because it is surface treatment applied
to a shape that is about to change. Texturing the wrong roof is wasted work.
Exception: the grass-lawn plan is **dead** — his houses sit on checkerboard
pavers, which is now photographically settled.
